Settings

Paths and tier. API keys live in ~/.job_runner/.env (not shown here). Keyword ↔ résumé mappings and uploaded files live under app_dir below (job_interests_yaml, role_resumes_dir) — they are local unless you point JOB_RUNNER_DIR at a cloud-synced folder (OneDrive, iCloud Drive, Dropbox, etc.) or back up those paths when switching machines.

Dropbox (or any sync folder) — no API needed

Install the Dropbox desktop app on the Mac and the PC (free tier is fine). Create a folder in Dropbox, e.g. JobRunnerData. On each machine, set environment variable JOB_RUNNER_DIR to that folder’s full path (Mac and Windows paths differ). Restart the terminal / app. Then the Mac UI and the Windows CLI both use the same synced data — uploads on the Mac appear on Windows after Dropbox syncs. No Dropbox API keys or OAuth.

Windows (User or System env): JOB_RUNNER_DIR=C:\Users\You\Dropbox\JobRunnerData
Mac (shell profile or launchctl): export JOB_RUNNER_DIR=/Users/You/Library/CloudStorage/Dropbox/JobRunnerData (your Dropbox path may be ~/Dropbox/JobRunnerData depending on how Dropbox is installed.)

Role résumés

Uploaded files and the search keywords that use each file. If the UI runs on another computer (e.g. server on Windows, you on Mac), use a pack: on the machine that has the résumés, click Download résumé pack, copy the zip to the server (AirDrop, USB, shared folder), then Import pack there so keyword ↔ file mappings apply on that instance.

Or open this app’s URL from the Mac browser (same Wi‑Fi/VPN as the Windows host, e.g. http://192.168.x.x:PORT) and use Find jobs — uploads go to the server directly.

After code changes, use Restart server if the UI port is stuck or the app is stale.
